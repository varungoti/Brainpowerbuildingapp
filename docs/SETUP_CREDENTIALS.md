# Setup: credentials & production configuration

Follow this once you have access to **Supabase**, **Razorpay**, and (optionally) **Sentry**. The app and Edge Function are already wired; you only add values.

## 1. Local web / Vite (`.env` in project root)

Copy from `.env.example`:

| Variable | Required | Purpose |
|----------|----------|---------|
| `VITE_SUPABASE_PROJECT_ID` | Yes for payments & AI | Full URL `https://YOUR_REF.supabase.co` or project ref |
| `VITE_SUPABASE_ANON_KEY` | Yes | Supabase **anon** key (Dashboard → Settings → API) |
| `VITE_SENTRY_DSN` | No | Client error reporting (production) |
| `VITE_APP_ENV` | No | `production` / `staging` for Sentry |
| `VITE_ADMIN_EMAILS` | No | Comma-separated emails allowed to open Blueprint |
| `VITE_ANALYTICS_ENDPOINT` | No | See `docs/PRODUCT_ANALYTICS.md` |
| `VITE_FEATURE_FLAGS` | No | Build-time flags, e.g. `payments_remote_kill` |

Never commit `.env`. `VITE_APP_VERSION` is injected from `package.json` at build time.

## 2. Supabase Edge Function secrets

Deploy the function in `supabase/functions/server/` (your project’s deploy flow / Dashboard).

Set **secrets** (Dashboard → Edge Functions → Secrets, or CLI):

| Secret | Required for | Purpose |
|--------|----------------|---------|
| `RAZORPAY_KEY_ID` | Live checkout | Razorpay key id |
| `RAZORPAY_KEY_SECRET` | Live checkout | Razorpay secret (verify signature) |
| `OPENAI_API_KEY` | Live AI (optional) | If unset, AI Counselor returns **demo** JSON |
| `ALLOWED_ORIGINS` | Production web | Comma-separated origins, e.g. `https://app.yoursite.com` |
| `REMOTE_CONFIG_JSON` | No | Runtime flags JSON, e.g. `{"payments_remote_kill":false,"ai_counselor_paused":false}` |

### Remote flags (runtime, no app rebuild)

- **HTTP:** `GET /functions/v1/make-server-76b0ba9a/remote-config` (also mounted at `/remote-config` for Supabase path stripping).
- **Merge order:** defaults → `REMOTE_CONFIG_JSON` env → KV key `remote_config:flags` (if you set it via tooling).
- **Supported flags:** `payments_remote_kill`, `ai_counselor_paused` (booleans).

## 3. GitHub Actions (optional)

Repository **Secrets**:

- `VITE_ADMIN_EMAILS` — team emails for admin Blueprint in CI builds (can be empty).

## 4. Capacitor / native

- Same `VITE_*` values are baked in at **`pnpm run build`** before `cap sync`.
- Rebuild and sync after any env change.
- Android: release signing and `versionCode` — see `capacitor.config.ts` and `docs/OWNER_ACTION_CHECKLIST.md`.

## 5. Razorpay dashboard

- Enable **Orders API** / standard checkout for your key mode (test vs live).
- Configure **webhook** (optional if you only verify in-app; your flow already verifies via Edge Function).
- Match **currency** (INR) and **key mode** (test keys for staging).

## 6. Verify locally

```bash
pnpm run build
pnpm run test:e2e          # default flows + paywall “not configured”
pnpm run test:e2e:paywall  # full mocked checkout (uses .env.e2e-paywall)
```

## 7. Autogenerated `utils/supabase/info.tsx`

If your toolchain generates this file from Figma/Make, keep it in sync with `.env` or remove it in favor of env-only config so you do not duplicate keys.

After filling secrets, complete items in **`docs/OWNER_ACTION_CHECKLIST.md`**.
